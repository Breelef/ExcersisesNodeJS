REST API

A REST API (Representational State Transfer Application Programming Interface) is a type of web service that uses the HTTP protocol to allow client applications to communicate with servers and perform various operations on data. RESTful APIs are commonly used for building web applications, mobile apps, and other client-server applications that require data to be accessed and manipulated over the internet.

URL Breakdown


Protocol:
Http is the protocol used for communication between the server and the client

Host:
this element identifies the domain name or IP address of the server that the client wants to communicate with.

Port:
this element specifies the port number that the client should use to communicate with the server. If no port is specified, the client will use the default port for the protocol (Port 80 for HTTP for example.)

Path:
This element specifies the path to the resource that the client wants to access on the server

Query String:
This element provides additional information to the server about the request. It is typically used to pass parameters to the server, such as search terms or filter criteria


XSS - Cross-site scripting

XSS (Cross-Site Scripting) is a type of web application security vulnerability that occurs when an attacker injects malicious code into a web page, which is then executed by the client's browser. This can allow the attacker to steal sensitive information, such as login credentials or personal data, or to perform unauthorized actions on behalf of the user

The way we have combated XSS, is when you manipulate a DOM and want to insert something, then we use innerText and not innerHTML. When you use innerHTML, you can insert scripts that can then run. By creating the element and then inserting the text into the element, we can avoid XSS 

                
                    const tankName = document.createElement("p");

                    tankName.innerText = tank.name;
